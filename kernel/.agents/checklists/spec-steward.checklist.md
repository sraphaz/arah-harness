# Checklist — Spec Steward

- [ ] Ambiguidades resolvidas antes de ativar spec
- [ ] Acceptance em formato EARS (`when`/`then`) quando aplicável
- [ ] `covered_by` ou `evidence` para AC com status covered/manual
- [ ] Harness commands válidos e executáveis
- [ ] Spec curta (1–3 páginas); overflow em `docs/specs/features/`
