Arquitetura da solução · modelo C4

Um frontend plugável, somente leitura, com os artefatos do harness como contrato

O console nunca é fonte de verdade: ele observa .arah/ e docs/_meta/ e projeta esse estado em tempo real. Remove-lo não muda nada no funcionamento do harness.

DECISÃO 01
Somente leitura por padrão
O console nunca escreve no repositório. Ações (aprovar, mesclar) são links para o PR — a autoridade continua onde sempre esteve.
DECISÃO 02
Artefatos como contrato de dados
Qualquer frontend que leia .arah/ e docs/_meta/ é compatível. O console é substituível; os arquivos, não.
DECISÃO 03
Índice descartável
O SQLite é cache: apagou, regenera dos JSONL. Resolve consulta e agregação sem criar segunda fonte de verdade.
DECISÃO 04
Três formas de implantação
Extensão de IDE (dev individual), app local (tech lead) e serviço de organização (multi-repo, futuro) — mesmo código, mesmos contratos.
