The shift

Software engineering is becoming human-agent engineering

The work is no longer just writing code. It is coordinating humans, agents, models, tools, policies, context, validation and persistent memory — as one system of engineering.

The future of software engineering is not autonomous. It is coordinated.

The problem

Code generation is not the hardest problem

The harder problem is coordinating intelligence without losing context, authority and accountability.

P·01
Agents without organizational context
Models read code locally but do not know responsibilities, architecture, domain boundaries or decision criteria.
P·02
Ephemeral prompts
Knowledge produced in AI sessions disappears when the conversation ends or the tool changes.
P·03
Autonomy without accountability
Agents execute changes with no explicit authority, scope, approval path or evidence.
P·04
Knowledge locked to vendors
Context and workflows get trapped inside copilots, proprietary platforms and specific runtimes.
P·05
Policies that are only documents
Rules live in wikis and PDFs, with no executable form an agent is forced to obey.
P·06
Uncoordinated multi-agent activity
Multiple agents act on the same repository without roles, boundaries or shared signals.
P·07
Absence of evidence
Changes land without artifacts proving what was validated, by whom, under which policy.
P·08
Context lost between sessions
Every new session starts from zero: decisions, constraints and history must be re-explained.
The inversion

The repository governs the agent.

The agent does not receive unrestricted authority. The repository provides its role, context, scope, capabilities, policies, validations and approval paths.

Agent-first
Prompt→ Agent→ Tools→ Code change

Authority flows from the conversation. Context lives in the session. Nothing persists.

Repository-first · ARAH
Human intent→ Repo context→ Domain→ Agents→ Skills→ Gates→ Evidence→ Pull request

Authority, scope and validation are versioned contracts in the repository. Everything persists.

Principles

Eight commitments the harness enforces

Repository-first
The repository is the operational source of truth.
Human in command
Accountability and merge authority remain human.
Governed agency
Every agent has an explicit role, scope and autonomy level.
Spec before code
Relevant changes start from intent and verifiable criteria.
Context on demand
Each agent receives the context appropriate to its function.
Evidence before trust
Confidence is sustained by validations and artifacts.
Persistent intelligence
Knowledge survives sessions, tools and models.
Evolution by selection
The system proposes; humans select.

Quick start

Install it in an existing repository

Three commands to install the kernel, then a small CLI surface to discover, sync, validate and evolve. Full guide in the documentation.

arah doctor
arah discover
arah domain sync
arah export-graph
arah evolve
1 · CLONE THE HARNESS
git clone https://github.com/sraphaz/arah-harness.git $env:USERPROFILE\arah-harness
2 · INSTALL INTO YOUR PROJECT
cd C:\path\to\your-project

powershell -ExecutionPolicy Bypass `
  -File $env:USERPROFILE\arah-harness\cli\arah.ps1 `
  install `
  -ProjectName "my-project"
3 · REGENERATE / UPDATE
powershell -File `
  $env:USERPROFILE\arah-harness\cli\arah.ps1 `
  regenerate `
  -Target . `
  -UpdateKernel
